#  实时数仓-分层 [1]

### 数据源

### 明细层

+ 目的是给下游提供直接可用的数据
+ 要对基础层进行统一的加工，比如清洗、过滤、扩维等
+ 按照主题进行管理

### 汇总层

+ 所有的指标都统一在汇总层加工
+ 汇总指标池
  按照统一的规范管理建设，形成可复用的汇总结果

# 实时数仓-构建流程 [1]

+ 搭框架

+ 定规范

+ 时效性
  设计的时候，层次不能太多
